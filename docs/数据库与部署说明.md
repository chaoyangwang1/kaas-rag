# KAAS-RAG 数据库与部署说明

---

## 一、数据库表结构总览

系统依赖以下数据存储：

| 存储 | 作用 | 数据库/集合 |
|---|---|---|
| MySQL | 知识库配置 + 文件记录 | `kaas_rag` |
| Milvus | 文档切片向量 | `kaas_chunks`、`kaas_item_names` |
| MongoDB | 会话历史 + 评估数据 | `kaas` |
| MinIO | 文件+图片对象存储 | `knowledge-base-files` |

---

## 二、MySQL 表结构

### 2.1 knowledge_bases（知识库配置表）

```sql
CREATE TABLE IF NOT EXISTS knowledge_bases (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    name            VARCHAR(200)    NOT NULL COMMENT '知识库名称',
    description     TEXT            COMMENT '知识库描述',
    config_json     TEXT            COMMENT '可调参数JSON',
    doc_count       INT             DEFAULT 0 COMMENT '已导入文档数',
    created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE INDEX idx_kb_name (name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='知识库表';
```

### 2.2 file_records（文件上传记录表）

```sql
CREATE TABLE IF NOT EXISTS file_records (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    kb_id           INT             NOT NULL DEFAULT 1 COMMENT '所属知识库ID',
    file_name       VARCHAR(500)    NOT NULL COMMENT '原始文件名',
    file_size       BIGINT          NOT NULL COMMENT '文件大小(字节)',
    file_type       VARCHAR(10)     NOT NULL COMMENT '文件类型: pdf/md',
    file_hash       VARCHAR(64)     NOT NULL DEFAULT '' COMMENT '文件MD5哈希',
    minio_bucket    VARCHAR(100)    NOT NULL COMMENT 'MinIO存储桶名',
    minio_object    VARCHAR(1000)   NOT NULL COMMENT 'MinIO对象路径',
    embedding_status VARCHAR(20)    NOT NULL DEFAULT 'uploaded' COMMENT 'uploaded/processing/completed/failed',
    task_id         VARCHAR(36)              COMMENT '关联任务ID(UUID)',
    error_message   TEXT                    COMMENT '失败时的错误信息',
    created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_kb_id (kb_id),
    INDEX idx_task_id (task_id),
    INDEX idx_status (embedding_status),
    INDEX idx_file_hash (file_hash),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文件上传记录表';
```

**embedding_status 状态流转：**

```
uploaded → processing → completed
                      → failed
```

## 三、Milvus 集合结构

### 3.1 kaas_chunks（文档切片集合）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` (主键) | INT64 | 自增主键 |
| `chunk_id` | VARCHAR | 切片业务 ID |
| `content` | VARCHAR | 切片文本内容 |
| `title` | VARCHAR | 切片标题（Markdown 标题层级） |
| `parent_title` | VARCHAR | 父级标题 |
| `file_title` | VARCHAR | 所属文件名（去后缀） |
| `item_name` | VARCHAR | 所属产品名称 |
| `kb_id` | INT64 | 所属知识库 ID |
| `task_id` | VARCHAR | 关联任务 ID |
| `part` | INT64 | 切片序号 |
| `dense_vector` | FLOAT_VECTOR(1024) | BGE-M3 稠密向量 |
| `sparse_vector` | SPARSE_FLOAT_VECTOR | BGE-M3 稀疏向量 |

**索引：**
- dense_vector: IVF_FLAT / HNSW（COSINE 相似度）
- sparse_vector: SPARSE_INVERTED_INDEX（IP 内积）

### 3.2 kaas_item_names（产品名称集合）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` (主键) | INT64 | 自增主键 |
| `item_name` | VARCHAR | 标准化产品名称 |
| `dense_vector` | FLOAT_VECTOR(1024) | BGE-M3 稠密向量 |
| `sparse_vector` | SPARSE_FLOAT_VECTOR | BGE-M3 稀疏向量 |

**用途：** 用于意图确认阶段的商品名向量检索，将用户提及的商品名与标准化名称进行匹配。

## 四、MongoDB 集合结构

### 4.1 chat_message（会话历史集合）

数据库：`kaas`，集合：`chat_message`

| 字段 | 类型 | 说明 |
|---|---|---|
| `_id` | ObjectId | 自动生成主键 |
| `session_id` | String | 会话唯一标识 |
| `role` | String | 消息角色：user / assistant |
| `text` | String | 消息内容 |
| `rewritten_query` | String | 改写后的查询（仅 user 消息） |
| `item_names` | Array[String] | 关联的商品名称列表 |
| `image_urls` | Array[String] | 关联的图片 URL 列表 |
| `ts` | Float | Unix 时间戳（秒） |

**索引：**
```javascript
db.chat_message.createIndex({"session_id": 1, "ts": -1})
```

### 4.2 评估相关集合（数据库：kaas）

| 集合 | 用途 |
|---|---|
| `eval_datasets` | 评估数据集 |
| `eval_items` | 测试条目（question + expected_answer + relevant_chunk_ids） |
| `eval_tasks` | 评估任务与结果 |

## 六、完整初始化流程

### 6.1 环境准备

```bash
# 1. 克隆项目
git clone <repo_url>
cd kaas-rag

# 2. 安装 Python 依赖（推荐使用 uv）
uv sync
# 或传统方式
pip install -e .
```

### 6.2 启动所有外部服务

```bash
# ===== MySQL =====
docker run -d --name mysql-kaas \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -e MYSQL_DATABASE=kaas_rag \
  mysql:8.0

# ===== MongoDB =====
docker run -d --name mongo-kaas \
  -p 27017:27017 \
  -e MONGO_INITDB_ROOT_USERNAME=root \
  -e MONGO_INITDB_ROOT_PASSWORD=123456 \
  mongo:7

# ===== MinIO =====
docker run -d --name minio-kaas \
  -p 9000:9000 -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  minio/minio server /data --console-address ":9001"

# ===== Milvus =====
# 方式1：Docker Compose（推荐）
wget https://github.com/milvus-io/milvus/releases/download/v2.4.0/milvus-standalone-docker-compose.yml -O docker-compose.yml
docker compose up -d

# 方式2：单容器
docker run -d --name milvus-standalone \
  -p 19530:19530 -p 9091:9091 \
  milvusdb/milvus:latest

# ===== BGE-Reranker（Docker 远程服务） =====
docker run -d --name reranker \
  -p 18081:80 \
  --gpus all \
  modelscope/bge-reranker-large:latest

# ===== BGE-M3 Embedding（Docker 远程服务，可选） =====
docker run -d --name embedding \
  -p 18088:80 \
  --gpus all \
  modelscope/bge-m3:latest
```

### 6.3 配置环境变量

```bash
# 复制示例配置
cp .env.example .env

# 编辑 .env 文件，配置以下关键项：
#   OPENAI_API_KEY      - LLM API 密钥（DeepSeek 或 DashScope）
#   OPENAI_BASE_URL     - LLM API 地址
#   MILVUS_URL          - Milvus 连接地址
#   MONGO_URL           - MongoDB 连接地址
#   MYSQL_HOST/PORT/USER/PASSWORD/DATABASE
#   MINIO_ENDPOINT      - MinIO 地址
#   MINERU_API_TOKEN    - MinerU PDF 解析 Token
#   RERANKER_BASE_URL   - Reranker 服务地址
#   EMBEDDING_USE_LOCAL - 是否使用本地 Embedding
```

### 6.4 启动应用（自动建表）


## 九、生产环境部署建议

### 配置调整

```bash
# .env 生产环境配置示例
# ===== 安全 =====
OPENAI_API_KEY=<使用密钥管理服务>
MINERU_API_TOKEN=<使用密钥管理服务>

# ===== Embedding 生产模式 =====
EMBEDDING_USE_LOCAL=false           # 使用远程 Docker 服务
EMBEDDING_BASE_URL=http://embedding:18088/v1
EMBEDDING_DEVICE=cuda:0             # GPU 加速

# ===== 日志 =====
LOG_CONSOLE_ENABLE=False            # 生产环境关闭控制台日志
LOG_FILE_LEVEL=WARNING              # 只记录警告及以上级别
LOG_FILE_RETENTION=30 days

# ===== 性能 =====
LLM_DEFAULT_TEMPERATURE=0.0         # 生产环境更确定性
```

### Docker 部署

```dockerfile
# Dockerfile 示例
FROM python:3.11-slim
WORKDIR /app
COPY . .
RUN pip install uv && uv sync
EXPOSE 8888
CMD ["python", "app/main.py"]
```

### Nginx 反向代理

```nginx
server {
    listen 443 ssl;
    server_name kaas-rag.example.com;

    location / {
        proxy_pass http://127.0.0.1:8888;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;  # SSE 流式需要关闭缓冲
        proxy_cache off;
    }
}
```
