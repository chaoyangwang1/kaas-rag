# KAAS-RAG 项目说明

## 项目定位

**KAAS-RAG**（Knowledge As A Service - RAG）是一个基于 LangGraph 的 RAG（检索增强生成）知识库系统，面向产品手册（PDF/Markdown）的智能问答场景。

### 当前阶段

当前工程已完成 RAG 核心模块的功能开发，用于技术沉淀和快速验证学习。经过实际验证，系统满足**生产级的高精度检索需求**。

### 生产化注意事项

如要投入生产环境使用，需要根据实际场景补充以下能力：

1. **接口鉴权**：当前 API 未做身份认证和鉴权（CORS 允许所有来源）。生产环境需增加 API Key / JWT / OAuth2 等认证机制。
2. **数据越权校验**：当前按 `kb_id` 进行知识库隔离，但未校验用户是否有权限访问该知识库。生产环境需增加用户→知识库的权限映射关系。
3. **性能优化**：当前为单进程内存模式（任务状态、Embedding 缓存均在进程内存中），生产环境需考虑：
   - 改用 Redis 替代内存字典存储任务状态（支持多进程/多实例）
   - Embedding 服务独立部署为 Docker 微服务并水平扩展
   - 大文件上传使用分块上传 + 断点续传
   - LLM 调用增加限流和熔断
4. **安全性**：`.env` 中的密钥需要改为密钥管理服务（如 Vault）；文件上传需增加类型和大小白名单校验。
5. **监控告警**：需接入 Prometheus + Grafana 监控各节点耗时、检索召回率、LLM 调用成功率等指标。

## 核心功能

### 1. 知识库导入

- 支持 PDF（通过 MinerU API 转换）和 Markdown 文件直接导入
- 自动文档分块（按 Markdown 标题层级 + 长度限制）
- Markdown 图片自动提取并上传到 MinIO
- BGE-M3 向量化（稠密 1024 维 + 稀疏词级向量）
- 向量写入 Milvus 向量数据库
- 主体名称（产品名）自动识别与入库
- 文件去重（MD5）+ 覆盖上传 + 级联删除

### 2. 智能问答

- 多轮对话支持（MongoDB 存储会话历史）
- 商品名自动识别与确认（支持反问交互）
- 查询改写（口语→规范查询）
- 4 路并行搜索：
  - 标准向量搜索（BGE-M3 稠密+稀疏混合）
  - HyDE 假设文档搜索
  - 知识图谱查询（Neo4j）
  - 网络搜索（DashScope MCP WebSearch）
- RRF 倒数排名融合 + BGE-Reranker 精排 + 动态 TopK 断崖截断
- 支持同步和 SSE 流式两种输出模式
- 实时任务进度追踪
- 图片 URL 自动提取并返回前端

### 3. 知识库管理

- 多知识库 CRUD，独立配置参数
- 知识库级删除保护
- 30+ 可调参数 JSON 配置，API 动态管理

### 4. 检索评估

- 数据集管理（JSON/CSV 批量导入）
- 自定义评估引擎 + RAGAS 标准评估
- 多组参数横向对比
- 后台异步执行

## 技术栈

| 类别 | 技术 | 说明 |
|---|---|---|
| Web 框架 | FastAPI | HTTP API + SSE 流式 |
| 工作流引擎 | LangGraph | 有状态 DAG 工作流（Send API 并行） |
| LLM 适配 | LangChain ChatOpenAI | 兼容 OpenAI/千问/DeepSeek 等 |
| Embedding | BGE-M3 (FlagEmbedding) | 稠密 1024D + 稀疏词级向量 |
| 向量数据库 | Milvus | 混合搜索（COSINE + IP） |
| 重排序 | BGE-Reranker-large | Docker 远程服务 |
| PDF 解析 | MinerU API | PDF → Markdown |
| 对象存储 | MinIO | 文件/图片存储 |
| 关系数据库 | MySQL | 知识库配置 + 文件记录 |
| 文档数据库 | MongoDB | 会话历史 + 评估数据 |
| 图数据库 | Neo4j | 知识图谱 |
| 日志 | Loguru | 文件+控制台，自动滚动 |
| 依赖管理 | uv | Python 依赖锁定 |

## 快速开始

### 1. 环境准备

```bash
# Python >= 3.11
# 安装 uv 包管理器
pip install uv

# 安装依赖
uv sync
```

### 2. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 文件，填入实际的 API Key 和服务地址
```

### 3. 确保外部服务就绪

```bash
# Milvus
docker run -d --name milvus-standalone -p 19530:19530 -p 9091:9091 milvusdb/milvus:latest

# MySQL
docker run -d --name mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 mysql:8.0

# MongoDB
docker run -d --name mongo -p 27017:27017 -e MONGO_INITDB_ROOT_USERNAME=root -e MONGO_INITDB_ROOT_PASSWORD=123456 mongo:7

# MinIO
docker run -d --name minio -p 9000:9000 -p 9001:9001 -e MINIO_ROOT_USER=minioadmin -e MINIO_ROOT_PASSWORD=minioadmin minio/minio server /data --console-address ":9001"

# BGE-Reranker
docker run -d --name reranker -p 18081:80 modelscope/bge-reranker-large:latest

# BGE-M3 Embedding (远程模式时启用)
docker run -d --name embedding -p 18088:80 modelscope/bge-m3:latest
```

### 4. 启动应用

```bash
python app/main.py
# 访问 http://localhost:8888
```

### 5. 下载模型（本地 Embedding 模式）

```bash
# 首次使用本地模式时，需要下载 BGE-M3 模型
python app/tool/download_bgem3.py
```
