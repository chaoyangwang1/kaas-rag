# KAAS-RAG 架构设计文档

> 基于 LangGraph 的 RAG（检索增强生成）知识库系统，面向产品手册的智能问答平台。

---

## 二、核心工作流详解

### 2.1 知识库导入流程 (`app/import_process/agent/main_graph.py`)

```
┌─────────────┐
│  START      │
└──────┬──────┘
       ▼
┌─────────────────┐
│   node_entry    │  ← 入口：初始化参数、校验输入、判断文件类型
│  (检查文件)      │
└────────┬────────┘
         │ 条件路由 route_after_entry()
         ├─── is_md_read_enabled ──────────────────────────────┐
         ├─── is_pdf_read_enabled ───┐                         │
         └─── 其他 → END             │                         │
                                     ▼                         │
                          ┌──────────────────┐                 │
                          │ node_pdf_to_md   │                 │
                          │ (MinerU API)     │                 │
                          │ PDF → Markdown   │                 │
                          └────────┬─────────┘                 │
                                   │                           │
                                   ▼                           ▼
                          ┌────────────────────────────────────────┐
                          │         node_md_img                    │
                          │  (MD图片处理: 提取/下载/修复图片路径)      │
                          └──────────────────┬─────────────────────┘
                                             ▼
                          ┌────────────────────────────────────────┐
                          │     node_document_split                │
                          │  (Markdown结构切分: 按标题层级+长度)      │
                          └──────────────────┬─────────────────────┘
                                             ▼
                          ┌────────────────────────────────────────┐
                          │   node_item_name_recognition           │
                          │  (LLM识别文档对应的产品/主体名称)         │
                          └──────────────────┬─────────────────────┘
                                             ▼
                          ┌────────────────────────────────────────┐
                          │      node_bge_embedding                │
                          │  (BGE-M3 生成稠密(1024维)+稀疏向量)     │
                          └──────────────────┬─────────────────────┘
                                             ▼
                          ┌────────────────────────────────────────┐
                          │      node_import_milvus                │
                          │  (向量数据批量写入Milvus集合)            │
                          └──────────────────┬─────────────────────┘
                                             ▼
                                        ┌────────┐
                                        │  END   │
                                        └────────┘
```

**关键代码文件：**
| 文件 | 功能 |
|---|---|
| `app/import_process/agent/state.py` | `ImportGraphState` 状态定义，24个字段 |
| `app/import_process/agent/nodes/node_entry.py` | 入口：文件类型判断，设置分支开关 |
| `app/import_process/agent/nodes/node_pdf_to_md.py` | 调用 MinerU API 将 PDF 转 Markdown |
| `app/import_process/agent/nodes/node_md_img.py` | 提取 MD 中的图片 URL，下载到 MinIO |
| `app/import_process/agent/nodes/node_document_split.py` | 按 Markdown 标题层级切分文档（chunk_max=2000, chunk_min=500） |
| `app/import_process/agent/nodes/node_item_name_recognition.py` | LLM 识别文档对应产品名称 |
| `app/import_process/agent/nodes/node_bge_embedding.py` | BGE-M3 向量化（稠密1024维 + 稀疏向量） |
| `app/import_process/agent/nodes/node_import_milvus.py` | 向量数据写入 Milvus 的 `kaas_chunks` 集合 |

### 2.2 智能问答查询流程 (`app/query_process/agent/main_graph.py`)

```
┌─────────────┐
│   START     │
└──────┬──────┘
       ▼
┌──────────────────────────────┐
│  node_item_name_confirm      │  ← 意图确认（7步子流程）
│  (商品名确认 + 查询改写)       │
│                              │
│  Step1: 获取历史对话           │
│  Step2: 保存用户消息           │
│  Step3: LLM提取商品名+改写     │
│  Step3.5: 反问回复匹配检测     │
│  Step4: Milvus向量检索商品名   │
│  Step5: 置信度对齐            │
│  Step6: 分支决策              │
│  Step7: 写入历史              │
└────────┬─────────────────────┘
         │ 条件路由 route_after_item_confirm()
         ├── answer已存在（反问/拒答）→ node_answer_output → END
         ├── enable_query_rewrite ──┐
         └── 直接多路搜索 ──────────┤
                                   ▼
                          ┌──────────────────┐
                          │node_query_rewrite│
                          │ (LLM查询改写优化) │
                          └────────┬─────────┘
                                   ▼
                   ┌───────────────────────────────┐
                   │    node_multi_search           │  ← 虚拟节点（分叉点）
                   │    Send API 并行分叉            │
                   └──┬────────┬────────┬──────────┘
                      │        │        │
          ┌───────────┘  ┌─────┘  ┌─────┘  └───────────┐
          ▼               ▼        ▼                     ▼
┌─────────────────┐ ┌───────────┐ ┌──────────┐ ┌────────────────┐
│node_search_     │ │node_search│ │node_query│ │node_web_search │
│embedding        │ │_embedding │ │_kg       │ │_mcp            │
│(向量搜索)        │ │_hyde      │ │(KG查询)  │ │(网络搜索MCP)    │
│BGE-M3→Milvus    │ │LLM假设文档 │ │Neo4j     │ │DashScope MCP   │
│混合搜索(D+S)     │ │→混合搜索  │ │实体查询   │ │WebSearch       │
└────────┬────────┘ └─────┬─────┘ └────┬─────┘ └───────┬────────┘
         │                │            │                │
         └────────┬───────┴────────────┴────────────────┘
                  ▼
          ┌─────────────┐
          │  node_join  │  ← 虚拟节点（合并点）
          └──────┬──────┘
                 ▼
          ┌────────────────┐
          │   node_rrf     │  ← RRF倒数排名融合
          │  k=60, max=10  │     各路搜索结果加权融合排序
          └───────┬────────┘
                  ▼
          ┌──────────────────────────────┐
          │        node_rerank            │
          │  Step1: 合并本地+Web文档       │
          │  Step2: BGE-Reranker重排序     │
          │  Step3: 动态TopK断崖截断       │
          └──────────────┬───────────────┘
                         ▼
          ┌──────────────────────────────┐
          │    node_answer_output         │
          │  Step1: 检查已有answer         │
          │  Step2: 构建Prompt模板         │
          │  Step3: LLM生成答案(流式/同步)  │
          │  Step4: 提取图片URL            │
          │  Step5: 写入MongoDB历史        │
          │  Step6: SSE推送最终事件         │
          └──────────────┬───────────────┘
                         ▼
                     ┌──────┐
                     │ END  │
                     └──────┘
```

**关键代码文件：**
| 文件 | 功能 |
|---|---|
| `app/query_process/agent/nodes/node_item_name_confirm.py` | 7步商品名确认流程（历史获取→LLM提取→向量检索→置信度对齐→分支决策） |
| `app/query_process/agent/nodes/node_query_rewrite.py` | LLM 将口语化查询改写为规范的检索查询 |
| `app/query_process/agent/nodes/node_search_embedding.py` | 标准向量搜索（BGE-M3 稠密+稀疏混合搜索） |
| `app/query_process/agent/nodes/node_search_embedding_hyde.py` | HyDE 假设文档检索（LLM生成虚拟答案→向量化检索） |
| `app/query_process/agent/nodes/node_query_kg.py` | Neo4j 知识图谱实体查询 |
| `app/query_process/agent/nodes/node_web_search_mcp.py` | DashScope MCP 联网搜索 |
| `app/query_process/agent/nodes/node_rrf.py` | 加权倒数排名融合（带权重的多路结果融合） |
| `app/query_process/agent/nodes/node_rerank.py` | BGE-Reranker 精排 + 动态 TopK 断崖截断 |
| `app/query_process/agent/nodes/node_answer_output.py` | Prompt构建→LLM生成→历史写入→SSE推送 |

## 四、API 接口交互逻辑

### 4.1 知识库导入接口 (`/`)

| 方法 | 路径 | 功能 | 说明 |
|---|---|---|---|
| `GET` | `/` | 返回前端页面 | `app/page/index.html` |
| `POST` | `/upload` | 文件上传 | multipart/form-data，支持 PDF/MD，后台异步执行导入流程 |
| `POST` | `/upload/overwrite` | 覆盖上传 | 删除旧文件的 MinIO/Milvus 数据后重新导入 |
| `GET` | `/status/{task_id}` | 任务状态查询 | 返回 done_list / running_list / status |
| `GET` | `/files` | 文件列表 | 支持 kb_id 过滤 |
| `DELETE` | `/files/{file_id}` | 删除文件 | 级联删除 MinIO + Milvus + MySQL |

**上传流程：**
```
POST /upload (文件 + kb_id)
  → 本地保存 → MD5去重检查
  → MySQL写入记录 → MinIO上传
  → BackgroundTasks.add_task(run_graph_task)
    → import_service.run_graph_task()
      → kb_import_app.stream(init_state)  ← LangGraph 导入工作流
      → 状态更新(processing→completed/failed)
```

### 4.2 智能问答接口 (`/query`)

| 方法 | 路径 | 功能 | 说明 |
|---|---|---|---|
| `POST` | `/query` | 提问 | 支持同步和流式两种模式 |
| `GET` | `/stream/{session_id}` | SSE流式获取 | 实时推送答案和进度 |
| `GET` | `/history/{session_id}` | 历史查询 | MongoDB 会话历史 |
| `DELETE` | `/history/{session_id}` | 清除历史 | 删除指定会话所有记录 |
| `GET` | `/health` | 健康检查 | 服务可用性检测 |

**同步模式交互：**
```
POST /query {"query": "xxx", "session_id": "uuid"}
  → 直接调用 query_app.invoke(state)
  → 返回 {"answer": "...", "entity_options": [...]}
```

**流式模式交互：**
```
POST /query {"query": "xxx", "session_id": "uuid", "is_stream": true}
  → 返回 {"session_id": "uuid"}  （立即返回）
  → BackgroundTasks 执行 query_app.invoke(state)
  → 前端 GET /stream/{session_id} (SSE 连接)
    → event: progress  {"done_list": [...], "running_list": [...], "status": "..."}
    → event: delta     {"delta": "逐字..."}
    → event: final     {"answer": "...", "image_urls": [...], "status": "completed"}
```

### 4.3 知识库管理接口 (`/api/kb`)

| 方法 | 路径 | 功能 |
|---|---|---|
| `GET` | `/api/kb` | 获取所有知识库 |
| `POST` | `/api/kb` | 创建知识库 |
| `GET` | `/api/kb/{kb_id}` | 获取知识库详情 |
| `GET` | `/api/kb/{kb_id}/config` | 获取知识库配置 |
| `PUT` | `/api/kb/{kb_id}` | 更新知识库 |
| `DELETE` | `/api/kb/{kb_id}` | 删除知识库（有删除保护） |

### 4.4 检索评估接口 (`/eval`)

| 方法 | 路径 | 功能 |
|---|---|---|
| `GET/POST` | `/eval/datasets` | 数据集管理 |
| `POST` | `/eval/datasets/{id}/import` | CSV/JSON批量导入测试条目 |
| `POST` | `/eval/tasks` | 创建评估任务（支持 ragas 和自定义模式） |
| `GET` | `/eval/tasks/{id}` | 任务状态查询 |
| `GET` | `/eval/tasks/{id}/compare` | 多组参数横向对比 |
